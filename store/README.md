# Spoiler Shield - Chrome Web Store assets & submission

Everything needed to (re)build the store graphics and submit the extension. Self-contained and
reproducible: the source images, the real screenshots, the pure-Python tooling, a one-command rebuild
script, and the step-by-step submission process. No third-party libraries required.

> If you're picking this up after a long break: run `python3 store/tools/build_assets.py`, then upload
> the files in `store/assets/` plus `store/spoiler-shield-<version>.zip`, using the field map under
> **Submission** below. Paste-ready listing text (description, summary, single-purpose, permission justifications) is in `store/listing/`; the full review notes are in [`../STORE.md`](../STORE.md).

## Brand / visual identity (keep consistent)

- **Gradient:** indigo `#4F46E5` (top) → violet `#7C3AED` (bottom)
- **Accent:** rose-red `#F43F5E` (the "no" slash)
- **Light:** off-white `#F6F7FF` (clock, text)
- **Emblem:** glossy rounded badge, white analog clock (bold ring, hands at 12 & 3, no numbers), crossed by one bold rose-red diagonal prohibition slash (lower-left → upper-right) = "no time".
- **Font:** captions are Ubuntu **Bold**; the wordmark in the promo art is a generic bold sans.

This matches the shipped toolbar icon (`icons/icon128.png`, generated by `tools/make-icons.mjs`). The
internal CSS class prefix in the extension is `anticip-` - intentional, not a stray old name (see the
root README).

## How each asset is made

| Asset | Size | Built from | Operation |
| --- | --- | --- | --- |
| `store-icon-128.png` | 128×128 | `emblem-baseline-1254.png` | downscale |
| `marquee-1400x560.png` | 1400×560 | `promo-master-1536x1024.png` | center-crop to 2.5∶1, resize |
| `small-tile-440x280.png` | 440×280 | `promo-master-1536x1024.png` | center-crop to 1.571∶1, resize |
| `screenshot-1-hero` | 1280×800 | `promo-master-1536x1024.png` | center-crop to 1.6∶1, resize |
| `screenshot-2..6` | 1280×800 | `screenshot-frame` + a `raw-screenshots/*` | fit capture into the card, add caption |

All outputs are **24-bit PNG, no alpha** (the store rejects alpha on screenshots/tiles).

| Output | Caption |
| --- | --- |

## Regenerate the graphics

```bash
python3 store/tools/build_assets.py
```

- Requires only **python3** (stdlib). Writes `store/assets/` (final) and `store/backups/screenshots-nocaption/`.
- **Captions** need a bold TrueType font. The script searches, in order: `store/tools/caption-font.ttf`,
  Ubuntu-B, DejaVuSans-Bold, LiberationSans-Bold. If none is found it still produces the framed
  screenshots (un-captioned) - to guarantee captions on any machine, drop any bold `.ttf` at
  `store/tools/caption-font.ttf`.
- **Revert captions:** the un-captioned versions are written to `backups/screenshots-nocaption/`; copy
  the ones you want over the matching files in `assets/`.

### Regenerate the SOURCE images (if `sources/*.png` are ever lost)

The three `sources/*.png` were generated with an image model (e.g. ChatGPT / GPT-4o image). Prompts:

**Shared style block (prepend to each prompt):**
```
BRAND: "Spoiler Shield" - a browser extension that hides video durations/timers so you aren't spoiled.
PALETTE: gradient indigo #4F46E5 (top) -> violet #7C3AED (bottom); accent rose-red #F43F5E; off-white #F6F7FF.
EMBLEM: glossy rounded badge, white analog clock (bold ring, two hands, no numbers) crossed by ONE bold
diagonal rose-red prohibition slash. STYLE: modern flat vector, soft depth, premium, OPAQUE background,
no real YouTube/Twitch logos, minimal/no text unless specified.
```

**emblem-baseline** (square, ~1024-1254):
```
1024x1024 app icon. Rounded-square tile (corner radius ~22%), indigo->violet gradient. White clock centered
(~50% width, ~47% height): bold ring, no numbers, hands pointing UP (12) and RIGHT (3). ONE bold rose-red
diagonal slash from LOWER-LEFT to UPPER-RIGHT over the clock. Faint glossy top highlight. Solid opaque
background. No text. Must read at 16px.
```

**promo-master** (1536×1024) - *attach the emblem image*:
```
Use the EXACT emblem in the attached image; do not redesign. 1536x1024 horizontal promo banner, full-bleed,
opaque. indigo->violet gradient + soft glow + a few faint generic "video card" rectangles (NO real logos),
each with its corner time badge replaced by a small rose-red shield/blur and a greyed-out progress bar.
Emblem large, left-of-center. Right: wordmark "Spoiler Shield" bold white + tagline
"Hide video durations - never get spoiled." Keep emblem/text/cards in the central band; top ~200px and
bottom ~200px plain gradient (safe to crop). Spell text exactly.
```

**screenshot-frame** (1280×800-ish) - *attach the emblem image*:
```
Reuse the exact emblem, small, top-left. 1280x800 opaque background template: indigo->violet gradient with
soft glow, a large empty rounded "device frame" (subtle drop shadow) to hold a browser screenshot, and a
clear strip at top for a short headline. No fake UI inside the frame, no body text.
```

Then re-run `build_assets.py`. (The crop/caption logic auto-detects the card and emblem, so minor
size/position differences in regenerated sources are tolerated.)

## Rebuild the extension package (ZIP)

```bash
bash store/tools/build_zip.sh        # -> store/spoiler-shield-<version>.zip
```

It zips `manifest.json content icons popup` with `manifest.json` at the ZIP root (required) and names the
file from the manifest `version`. Bump `version` in `manifest.json` before each new upload.

## Submission (Chrome Web Store Developer Dashboard)

1. **Package** → upload `store/spoiler-shield-<version>.zip`.
2. **Store icon** → `assets/store-icon-128.png`.
3. **Screenshots** (max 5, 1280×800) - recommended set, in order:
   `screenshot-1-hero`, `screenshot-3-cs2`, `screenshot-2-youtube`, `screenshot-6-twitch-videos`,
   `screenshot-5-twitch-no-seekbar`. (`screenshot-4-chapters` is the spare.)
4. **Small promo tile** → `assets/small-tile-440x280.png`.
5. **Marquee promo tile** → `assets/marquee-1400x560.png`.
6. **Promo video** → optional, leave blank.
7. **Listing text** - paste from `listing/`: `single-purpose.txt`, `description.txt`, `summary.txt`
   (mirrors the manifest `description`), and `permission-justifications.txt`. Full rationale & review notes:
   [`../STORE.md`](../STORE.md).
8. **Privacy** - answer "not collected" for every data category, certify Limited Use, and host
   `listing/privacy-policy.md` (copy of [`../PRIVACY.md`](../PRIVACY.md)) at a public URL (e.g. a GitHub Gist)
   to paste in the **Privacy policy** field.

### Pre-submit checks
- Loads unpacked with no console errors on youtube.com and twitch.tv.
- Master toggle + each per-site sub-toggle works live (no reload).
- "Reveal" peeks on the current video and re-hides on navigation.
- Keyboard seeking works with the seekbar hidden (← / → and J/K/L).
- Listing text + permission justifications + privacy answers entered; privacy-policy URL set.

## Notes

- **Why custom Python tooling:** this machine had no ImageMagick/Pillow/pip, so `imgkit.py` (PNG) and
  `ttf.py` (TrueType text) are hand-rolled with the standard library only. They are intentionally minimal
  (8-bit, non-interlaced PNG; `glyf`-based TTF) - enough for these assets, not a general image library.
- **The store text and the extension code** are versioned in the repo root (`STORE.md`, `PRIVACY.md`,
  `manifest.json`, `content/`, `popup/`). This `store/` folder is only the listing-graphics pipeline.
